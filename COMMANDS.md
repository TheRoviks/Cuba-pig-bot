# Команды и запуск

В этом файле собраны практические вещи: установка, настройка `.env`, запуск бота и список доступных команд.

## Требования

- Python 3.11+
- Telegram bot token от `@BotFather`

## Установка

```bash
pip install -r requirements.txt
```

## Настройка `.env`

Создай рядом с `bot.py` файл `.env`.

Пример:

```env
BOT_TOKEN=твой_токен_от_botfather
DB_PATH=quotes.db
ADMIN_USER_IDS=123456789,987654321
```

### Что значит каждая переменная

- `BOT_TOKEN` — токен Telegram-бота
- `DB_PATH` — путь к SQLite-базе; можно оставить `quotes.db`
- `ADMIN_USER_IDS` — список глобальных админов через запятую

Если `BOT_TOKEN` пустой, бот не запустится.

## Запуск

```bash
python bot.py
```

При старте бот:

- читает `.env`
- создаёт и обновляет структуру SQLite
- включает `WAL` и `foreign_keys`
- поднимает polling
- начинает обновлять данные по чатам и пользователям

## Как пользоваться в группе

### Сохранить цитату

1. Ответь на сообщение.
2. Напиши `/save_quote`.
3. Бот сохранит цитату и вернёт её id.

### Показать случайную цитату

```text
/q
```

`/q`, `/quote` и `/quote_random` в этом проекте делают одно и то же: показывают случайную цитату из текущего чата.

### Найти цитату по id

```text
/quote_id 15
```

### Найти по тексту

```text
/search_quote компилируюсь
```

### Удалить цитату

```text
/delete_quote 15
```

Удалить может тот, кто сохранил цитату, или администратор группы.

## Групповые команды

- `/start` — короткое приветствие в группе
- `/help` — список групповых и приватных возможностей
- `/save_quote` — сохранить цитату из реплая
- `/q` — случайная цитата
- `/quote` — случайная цитата
- `/quote_random` — случайная цитата
- `/quote_user` — цитаты конкретного человека
- `/my_quotes` — последние цитаты, которые сохранил ты
- `/quotes_count` — сколько живых цитат в текущем чате
- `/top_quoted` — локальный топ цитируемых
- `/delete_quote <id>` — мягко удалить цитату по id
- `/search_quote <текст>` — поиск по тексту в рамках текущего чата
- `/latest_quotes [limit]` — последние цитаты чата
- `/quote_id <id>` — показать цитату по id в рамках текущего чата

## Команды в личке

- `/start` — открыть приветствие и мини-кабинет
- `/help` — справка по личке
- `/my_chats` — общие чаты с ботом
- `/my_quotes_global` — сводка твоих цитат по доступным чатам
- `/my_quotes_chat <номер или chat_id>` — твои цитаты из одного выбранного чата

### Примеры для лички

```text
/my_quotes_chat 1
/my_quotes_chat -1001234567890
```

## Команды глобальной админки

Эти команды работают только:

- в личке
- для `user_id`, которые есть в `ADMIN_USER_IDS`

Список команд:

- `/admin` — открыть админ-панель
- `/admin_chats` — все известные чаты
- `/admin_stats` — общая статистика по боту
- `/admin_chat <chat_id>` — детали одного чата
- `/admin_users` — глобальные топы пользователей
- `/admin_health` — технический статус базы и бота

## Что делает админка

- показывает общее число чатов, пользователей и цитат
- считает цитаты за последние 24 часа и 7 дней
- показывает топ чатов по количеству цитат
- показывает топ сохранявших и топ цитируемых
- умеет смотреть детали конкретного чата
- показывает health по базе, схеме и служебным логам

## Inline-кнопки

У бота есть кнопки:

- у цитат: показать ещё одну случайную цитату, удалить цитату
- в личке: открыть список чатов и свои цитаты
- в админке: открыть статистику, чаты, пользователей, health

Все callback-переходы дополнительно проверяются на серверной стороне.

## Что попадает в базу

Кроме самих цитат, бот пишет:

- пользователей в `users`
- чаты в `chats`
- связи пользователь ↔ чат в `user_chat_links`
- события цитат в `quote_events`
- использование команд в `usage_logs`
- действия глобальных админов в `admin_audit_logs`

## Если бот добавлен в новый чат

Когда бота добавляют в группу, он:

- приветствует чат
- объясняет, что собирает цитаты
- напоминает про отдельный цитатник этого чата
- просит выдать админку для более стабильной работы

Без админки бот всё равно работает.

## Полезные замечания

- Все обычные запросы по цитатам фильтруют только живые записи, где `is_deleted = 0`.
- Данные между чатами не смешиваются.
- Пользователь в личке не может открыть чужой чат просто по `chat_id`.
- Админские команды не работают в группах.
